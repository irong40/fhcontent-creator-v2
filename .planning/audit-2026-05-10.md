## SEV1 — Blocks Sunday's batch

### S1.1 — daily-topic 504 timeout
**File:** `src/app/api/cron/daily-topic/route.ts:84-327`
**Evidence:** Vercel runtime log 2026-05-10 02:00:36 GET /api/cron/daily-topic 504. Topic id `5e5d131d` stuck content_generating, no pieces.
**Cause:** 7 topics × (1 topic Claude call + 1 content Claude call) sequential ≈ 700-900s. maxDuration=800s. Last topic times out.
**Fix:** Split into two passes — Sunday cron generates 7 topic SHELLS only (status=draft); a separate hourly cron (or content-generation cron) drains drafts. OR keep single cron but move content-gen to background job.

### S1.2 — `partially_published` never retries
**File:** `src/app/api/cron/daily-publish/route.ts:250` and `check-status/route.ts:283-294`
**Evidence:** every recent topic frozen in `partially_published` with stale IG/Bluesky failures. Daily-publish queries only `['scheduled', 'approved']`.
**Fix:** Include `partially_published` in the picker. Existing per-platform retry logic (line 144) already skips platforms not in `failed` state, so re-running is safe.

### S1.3 — Bluesky 100% fail "No account configured"
**File:** `daily-publish/helpers.ts:14-18` (target platforms include `bluesky` for short_*); persona Dr. Carter has no `bluesky` key in `platform_accounts`.
**Fix:** Either (a) add the bluesky account_id to Dr. Carter's `platform_accounts`, or (b) remove bluesky from default short targets. Going with (b) until accounts exist — getTargetPlatforms returns only platforms whose account_id exists on the persona at runtime.

### S1.4 — Instagram 100% fail "max 5 hashtags"
**File:** `daily-publish/helpers.ts:60-68` (capInstagramHashtags) + `prompts.ts:163`
**Evidence:** Cap function works in isolation. But Blotato still rejects. Suspect off-by-one in Blotato's counter, or a stale-state retry loop. Combined with S1.2, no retry.
**Fix:** (a) Tighten cap to 4 (defensive); (b) tighten prompt to "use exactly 3 hashtags"; (c) S1.2 fix gives retries.

### S1.5 — No intra-day staggering
**File:** `daily-publish/route.ts:252` (`lte('publish_date', today)`)
**Evidence:** `publish_date` is DATE only; all of a day's pieces ship in one hourly cron run.
**Fix:** Add `publish_at timestamptz` column to topics. Daily-topic sets it to a per-day spread (e.g., 13:00 UTC). Daily-publish filters `lte('publish_at', now())`.

## SEV2 — High-likelihood future failures

### S2.1 — Operator precedence bug
**File:** `daily-media/route.ts:570-572`
**Cause:** `NEXT_PUBLIC_SITE_URL || VERCEL_URL ? https://${VERCEL_URL} : ...` — wrong precedence. If NEXT_PUBLIC_SITE_URL is set but VERCEL_URL isn't, becomes `https://undefined`.
**Fix:** Add parens.

### S2.2 — daily-media unbounded query
**File:** `daily-media/route.ts:114-118`
**Cause:** Pulls every `content_ready|approved|scheduled` topic regardless of date. Sunday batch = 7 topics × ~6 pieces × DALL-E + ElevenLabs + HeyGen = will time out.
**Fix:** Add `.lte('publish_date', tomorrow)`.

### S2.3 — Carousel silent-stuck
**File:** `daily-media/route.ts:444-518`
**Cause:** If all slides fail subject-audit, `imageUrls=[]`, carousel keeps `carousel_url=null`. No status update, no alert. Topic publishes other pieces but carousel sits forever.
**Fix:** Mark carousel piece `status=failed` with audit reason if all slides fail.

### S2.4 — No Blotato rate-limiting
**File:** `daily-publish/route.ts:129-179`
**Cause:** 25+ Blotato uploads/posts back-to-back. Cascading failures if Blotato rate-limits.
**Fix:** 300ms delay between platform pushes within a piece.

## SEV3 — Quality / drift

### S3.1 — Hashtag prompt vague
**File:** `prompts.ts:163` says "3-5 hashtags ... mix broad-reach, niche, and discovery"
**Reality:** Claude generates 18-20 hashtags every time.
**Fix:** Hard rule: "Use EXACTLY 3 hashtags, period."

### S3.2 — Draft-skip race
**File:** `daily-topic/route.ts:97-110`
**Cause:** If a topic is left in `draft` (zero pieces inserted earlier), the persona is skipped FOREVER until manual cleanup. Combined with S1.1 (today's 7th topic), this is sitting on a landmine.
**Fix:** Exclude `draft` and `content_generating` from the skip-set OR auto-fail any topic stuck in those states >24h before running daily-topic.

### S3.3 — `content_ready` is dead state
**File:** `daily-topic` jumps `content_generating → scheduled`, never sets `content_ready`. UI/queries depending on it count zero.
**Fix:** Either set `content_ready` between content-gen and schedule, OR remove from queries. Going with: remove from queries.

### S3.4 — Evergreen ignores AUTO_TOPIC_PERSONA_IDS
**File:** `evergreen.ts:32-37`
**Cause:** Evergreen fills gaps for ALL active personas, including ones not in autonomous mode. Could schedule unverified Masonic content (which has notebooklm guardrail).
**Fix:** Filter by AUTO_TOPIC_PERSONA_IDS env.

### S3.5 — Memory drift: `feedback_git-workflow.md`
**Says:** "Always work on dev, merge to main for production"
**Reality:** Production branch is `master`. `main` is stale (last commit Apr).
**Fix:** Update memory after this audit.
