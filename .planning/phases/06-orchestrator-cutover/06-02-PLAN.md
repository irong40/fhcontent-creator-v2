  }

After "Research Succeeded?" true branch:

1. Get WF-2 webhook URL from n8n:
   - Open WF-2 → Webhook node → copy URL: `https://n8n.faithandharmonyllc.com/webhook/wf2-media`

2. Add **HTTP Request** node named "Trigger WF-2 Media (Async)":
   - Method: POST
   - URL: `https://n8n.faithandharmonyllc.com/webhook/wf2-media`
   - Headers:
     - `x-webhook-secret`: `={{ $credentials["N8N Webhook Secret"].value }}`
     - `X-Correlation-ID`: `={{ $('Set Correlation Context').item.json.correlation_id }}`
     - `Content-Type`: `application/json`
   - Body:
```json
{
  "topic_id": "={{ $('Set Correlation Context').item.json.topic_id }}",
  "correlation_id": "={{ $('Set Correlation Context').item.json.correlation_id }}",
  "trigger_source": "wf1"
}
```
   - Timeout: 10000ms (WF-2 webhook responds immediately with 200)
   - Retry On Fail: ON, Max Tries: 3, Wait: 2000ms
   - **This is fire-and-forget**: WF-1 does NOT wait for WF-2 to complete

   Note: WF-2 webhook is on the same n8n instance — call goes through Cloudflare Tunnel and back. Alternatively, use **Execute Workflow** node with wait=OFF for true async within n8n (more reliable than HTTP roundtrip):
   - Add **Execute Workflow** node named "Start WF-2 (Fire-and-Forget)":
     - Workflow: "WF-2 Media Pipeline"
     - Wait for Sub-Workflow: OFF (fire-and-forget)
     - Input: topic_id, correlation_id, trigger_source="wf1"

   Prefer the Execute Workflow approach (no HTTP roundtrip, no auth required between internal workflows).
  </action>

  <name>Task 4: Post to n8n-topic-callback with HMAC and log completion</name>
  <files>n8n-workflows/wf1-topic-pipeline.json</files>
  <action>
After firing WF-2:

1. Add **Code** node named "Build Topic Callback Payload":
```javascript
const ctx = $('Set Correlation Context').item.json;
const researchResult = $('Run WF-3 Research (Sync)').item.json;

return [{
  json: {
    topic_id: ctx.topic_id,
    correlation_id: ctx.correlation_id,
    run_id: ctx.run_id,
    status: 'pipeline_started',
    research_complete: true,
    media_pipeline_fired: true,
    completed_at: new Date().toISOString()
  }
}];
```

2. Add **Code** node named "Sign Topic Callback":
```javascript
const secret = $credentials["N8N Signing Secret"].value;
const body = JSON.stringify($input.item.json);
const crypto = require('crypto');
const sig = crypto.createHmac('sha256', secret).update(body).digest('hex');

return [{
  json: {
    _payload: body,
    _hmac: `sha256=${sig}`,
    correlation_id: $input.item.json.correlation_id
  }
}];
```

3. Add **HTTP Request** "POST to Topic Callback":
   - POST to `https://qjpujskwqaehxnqypxzu.supabase.co/functions/v1/n8n-topic-callback`
   - Headers: X-N8N-Signature, X-Correlation-ID, Content-Type, Authorization (Supabase Service Role)
   - Body: `={{ $json._payload }}`
   - Retry On Fail: ON, Max Tries: 3, Wait: 2000ms

4. Add **HTTP Request** "Log Run Complete":
   - PATCH pipeline_runs set status=completed, completed_at=now

5. Connect error outputs from "Call Topic Generate Route" and "Run WF-3 Research (Sync)" to **Execute Workflow: WF-Error** with:
```json
{
  "workflow_id": "WF-1",
  "correlation_id": "={{ $('Set Correlation Context').item.json.correlation_id }}",
  "error": "={{ $json.error || $json.message }}"
}
```
