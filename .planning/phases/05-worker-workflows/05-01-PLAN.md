Build WF-4 (Publish Pipeline) in n8n. This workflow runs hourly, acquires a workflow lock, calls the existing /api/cron/daily-publish Next.js route, and writes publish confirmations per platform to the n8n-publish-callback Edge Function with HMAC signing. Failed publishes after retry exhaustion are inserted to the pipeline_dlq table.

In n8n UI, create workflow named "WF-4 Publish Pipeline".

1. Add **Schedule Trigger** node named "Hourly Schedule":
   - Trigger interval: Every 1 hour
   - Set output: `{ "trigger_source": "schedule" }`

2. Add **Webhook** node named "Manual Trigger Webhook":
   - HTTP Method: POST
   - Path: `wf4-publish`
   - Authentication: Header Auth → `x-webhook-secret` header
   - Response mode: Respond immediately (200 OK before processing)
   - Set output to include: `{ "trigger_source": "webhook", "topic_id": "={{ $json.body.topic_id }}" }`

3. Add **Merge** node named "Merge Triggers":
   - Mode: Merge by position
   - Connect both triggers to this node

4. Add **Code** node named "Set Run Context":
```javascript
const trigger = $input.item.json.trigger_source || 'schedule';
return [{
  json: {
    trigger_source: trigger,
    topic_id: $input.item.json.topic_id || null,
    run_id: $execution.id,
    workflow_id: 'WF-4',
    lock_key: 'wf4-publish-pipeline',
    started_at: new Date().toISOString()
  }
```

   - URL: `https://fhcontent-creator-v2.vercel.app/api/workflow-lock/acquire`
   - Header Auth: Vercel Cron Secret
   - Body: `{ "lock_key": "={{ $json.lock_key }}", "workflow_id": "={{ $json.workflow_id }}", "run_id": "={{ $json.run_id }}" }`
   - Retry On Fail: ON, Max Tries: 3, Wait: 2000ms

6. Add **IF** node named "Lock Acquired?":
   - True → proceed to Task 2
   - False → Code node "Already Running — Skip": `return [];`

<task type="auto">
  <name>Task 3: Fan out per-platform confirmations to n8n-publish-callback</name>
  <files>n8n-workflows/wf4-publish-pipeline.json</files>
  <action>
The daily-publish route returns an array of publish results per platform:
```json
{
  "success": true,
  "published": [
    { "platform": "instagram", "content_piece_id": "uuid", "post_id": "..." },
    { "platform": "youtube", "content_piece_id": "uuid", "post_id": "..." }
  ]
}
```

1. Add **Code** node named "Extract Published Items":
```javascript
const results = $input.item.json.published || [];
const runId = $('Set Run Context').item.json.run_id;

return results.map(item => ({
  json: {
    ...item,
    correlation_id: runId,
    published_at: new Date().toISOString()
  }
}));
```

2. Add **Code** node named "Sign Publish Callback":
```javascript
const secret = $credentials["N8N Signing Secret"].value;
const body = JSON.stringify($input.item.json);
const crypto = require('crypto');
const sig = crypto.createHmac('sha256', secret).update(body).digest('hex');

return [{
  json: {
    _payload: body,
    _hmac: `sha256=${sig}`,
    correlation_id: $input.item.json.correlation_id,
    platform: $input.item.json.platform
  }
}];
```

3. Add **HTTP Request** node named "POST to Publish Callback":
   - Method: POST
   - URL: `https://qjpujskwqaehxnqypxzu.supabase.co/functions/v1/n8n-publish-callback`
   - Headers:
     - `X-N8N-Signature`: `={{ $json._hmac }}`
     - `X-Correlation-ID`: `={{ $json.correlation_id }}`
     - `Content-Type`: `application/json`
     - `Authorization`: `Bearer {{ $credentials["Supabase Service Role"].value }}`
   - Body: `={{ $json._payload }}`
   - Retry On Fail: ON, Max Tries: 3, Wait: 2000ms
   - On Error: Continue to DLQ path

4. After successful callbacks, add **HTTP Request** node named "Release Lock":
   - Method: POST
   - URL: `https://fhcontent-creator-v2.vercel.app/api/workflow-lock/release`
   - Header: Vercel Cron Secret
   - Body: `{ "lock_key": "wf4-publish-pipeline" }`
   - Retry On Fail: ON, Max Tries: 5, Wait: 1000ms

5. Add **HTTP Request** node named "Log Run Complete":
   - PATCH to pipeline_runs, set status=completed, completed_at=now
