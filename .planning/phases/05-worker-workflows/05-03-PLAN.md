<objective>
Build WF-2 (Media Pipeline) in n8n. This is the most complex workflow — it fires at 6AM daily, acquires a lock, calls /api/cron/daily-media to get the day's topic, then fans out across 6+ parallel branches to initiate asset generation (HeyGen video, Blotato short-form videos for 4 platforms, thumbnail, carousel, podcast, music). Each branch writes back to n8n-asset-callback with HMAC signing. The design respects the "async sweeper" pattern: branches fire HTTP calls that return immediately; WF-5 polls for completion.
</objective>

<task type="auto">
  <name>Task 2: Create WF-2 with dual trigger and topic acquisition</name>
  <files>n8n-workflows/wf2-media-pipeline.json</files>
  <action>
In n8n UI, create workflow named "WF-2 Media Pipeline".

1. Add **Schedule Trigger** named "6AM Daily":
   - Cron expression: `0 6 * * *`
   - Output: `{ "trigger_source": "schedule" }`

2. Add **Webhook** node named "Manual Trigger Webhook":
   - Path: `wf2-media`
   - Method: POST
   - Header Auth: `x-webhook-secret`
   - Response: Respond immediately

3. Add **Merge** node "Merge Triggers".

4. Add **Code** node named "Set Run Context":
```javascript
return [{
  json: {
    trigger_source: $input.item.json.trigger_source || 'manual',
    topic_id: $input.item.json.topic_id || null,
    run_id: $execution.id,
    workflow_id: 'WF-2',
    lock_key: 'wf2-media-pipeline',
    correlation_id: $execution.id,
    started_at: new Date().toISOString()
  }
}];
```

5. Add **HTTP Request** "Acquire Workflow Lock" (same pattern as WF-4).

6. Add **IF** "Lock Acquired?" → true branch continues, false → `return [];`.

7. Add **HTTP Request** "Log Run Start" → pipeline_runs POST.

8. Add **HTTP Request** named "Call Daily-Media Route":
   - Method: POST
   - URL: `https://fhcontent-creator-v2.vercel.app/api/cron/daily-media`
   - Headers: `Authorization: Bearer <CRON_SECRET>`, `X-Correlation-ID: ={{ correlation_id }}`
   - Body: `{ "topic_id": "={{ $json.topic_id }}" }`
   - Timeout: 30000ms
   - Retry On Fail: ON, Max Tries: 3, Wait: 4000ms

9. Add **Code** node "Extract Topic Context":
```javascript
const dailyMedia = $input.item.json;
const ctx = $('Set Run Context').item.json;

// daily-media route returns the topic to process
const topicId = dailyMedia.topic_id || ctx.topic_id;
if (!topicId) throw new Error('WF-2: No topic_id returned from daily-media route');

return [{
  json: {
    topic_id: topicId,
    topic_data: dailyMedia.topic || {},
    correlation_id: ctx.correlation_id,
    run_id: ctx.run_id
  }
}];
```

<task type="auto">
  <name>Task 4: Write-back per branch to n8n-asset-callback with HMAC</name>
  <files>n8n-workflows/wf2-media-pipeline.json</files>
  <action>
After each branch HTTP Request node, add per-branch callback nodes. Use a shared sub-pattern:

For each branch (example for HeyGen — repeat for all 6):

1. Add **Code** node "Sign HeyGen Callback":
```javascript
const ctx = $('Extract Topic Context').item.json;
const result = $input.item.json;

const payload = {
  topic_id: ctx.topic_id,
  correlation_id: ctx.correlation_id,
  asset_type: 'heygen-video',
  job_id: result.job_id || result.id,
  status: result.success !== false ? 'initiated' : 'failed',
  initiated_at: new Date().toISOString(),
  error: result.error || null
};

const secret = $credentials["N8N Signing Secret"].value;
const body = JSON.stringify(payload);
const crypto = require('crypto');
const sig = crypto.createHmac('sha256', secret).update(body).digest('hex');

return [{ json: { _payload: body, _hmac: `sha256=${sig}`, correlation_id: ctx.correlation_id } }];
```

2. Add **HTTP Request** "POST HeyGen Callback to Asset Edge Fn":
   - POST to `https://qjpujskwqaehxnqypxzu.supabase.co/functions/v1/n8n-asset-callback`
   - Headers: X-N8N-Signature, X-Correlation-ID, Content-Type, Authorization (Supabase Service Role)
   - Body: `={{ $json._payload }}`
   - Retry On Fail: ON, Max Tries: 3, Wait: 2000ms
   - On Error: Continue → DLQ insert

Repeat this pair for each of the 6 branches, with `asset_type` set appropriately:
- `heygen-video`
- `blotato-instagram`
- `blotato-tiktok`
- `blotato-facebook`
- `static-assets`
- `audio-assets`

DLQ error handler for failed callbacks (same pattern as WF-4 Task 4):
- Format DLQ Entry Code node → HTTP Request to `pipeline_dlq`
- asset_type and job_id included in DLQ payload
